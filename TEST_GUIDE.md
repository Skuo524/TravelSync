# TravelSync 功能測試指南

本文件旨在引導測試人員驗證 **TravelSync 最小可行產品 (MVP)** 的核心功能。

---

## 1. 測試前準備
在開始測試前，請確保：
1. 已安裝必要套件：`pip install -r requirements.txt`
2. 啟動應用程式：`streamlit run app.py`
3. 系統將在瀏覽器開啟，網址通常為 `http://localhost:8501`

---

## 2. 核心功能測試案例

### 測試案例 01：使用者身份與行程建立
*   **步驟**：
    1. 在左側側邊欄「輸入您的暱稱」處填寫您的名字（例如：Yunotang）。
    2. 點擊「建立新行程」展開表單。
    3. 輸入旅遊名稱「東京之旅」、選擇下週的開始與結束日期。
    4. 點擊「確認建立」。
*   **預期結果**：
    *   頁面顯示成功訊息。
    *   主畫面出現「📍 東京之旅」標題。
    *   顯示正確的日期範圍與一個 8 碼的「邀請碼」。

### 測試案例 02：加入現有行程 (多人協作模擬)
*   **步驟**：
    1. 複製測試案例 01 中產生的「邀請碼」。
    2. 開啟瀏覽器的 **「無痕視窗 (Incognito)」** 並進入同一個網址。
    3. 在無痕視窗中輸入不同的暱稱（例如：Guest_User）。
    4. 點擊側邊欄「加入行程」，貼上剛才的邀請碼並點擊「確認加入」。
*   **預期結果**：
    *   兩個視窗皆顯示相同的「東京之旅」行程內容。

### 測試案例 03：時間軸與景點管理
*   **步驟**：
    1. 在「第 1 天」的分頁中，展開「➕ 新增第 1 天行程項目」。
    2. 輸入名稱「淺草寺」、設定時間為 09:00、輸入地點「東京都台東區」。
    3. **不勾選**「這是一個提案」，點擊「新增到行程」。
*   **預期結果**：
    *   時間軸出現藍色標籤卡片，顯示「🕒 09:00」與「📍 淺草寺」。
    *   另一個瀏覽器視窗重新整理後（或操作後）應能看到相同項目。

### 測試案例 04：投票機制與提案定案
*   **步驟**：
    1. 在「第 1 天」分頁再次新增項目。
    2. 輸入名稱「午餐：燒肉 vs 壽司」、**勾選**「這是一個提案」。
    3. 點擊「新增到行程」。
    4. 在視窗 A 點擊該卡片上的「➕ 投票」。
    5. 切換到視窗 B，重新整理後點擊同一個項目的「➕ 投票」。
    6. 由其中一位使用者點擊「✅ 採納」。
*   **預期結果**：
    *   提案卡片顯示「🗳️ 提案中」，並統計票數為 2。
    *   點擊「採納」後，該卡片轉變為正式行程（黃色警告標籤變為藍色資訊標籤）。

### 測試案例 05：刪除功能
*   **步驟**：
    1. 點擊任何一個行程卡片右上角的「🗑️ 刪除」按鈕。
*   **預期結果**：
    *   該項目立即從畫面中消失。

## 3. 自動化邏輯驗證
除了手動測試，您也可以執行我們先前準備的測試腳本來驗證後端資料處理：
```cmd
python test_logic.py
```
**預期結果**：終端機應顯示 `✅ 所有功能驗證完成！`。

---

## 4. 常見問題排除
*   **資料沒有同步？**：Streamlit 是基於 Python 的前端，雖然具備反應式更新，但有時需手動操作（如點擊按鈕或重新整理頁面）觸發重新渲染。
*   **找不到資料庫？**：系統會自動在根目錄產生 `travelsync.db` 檔案，若要重置測試數據，直接刪除該檔案即可。
